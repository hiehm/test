+ 前言: +

+ 讓開發 Web 前端應用程式的過程中，完全排除手動操作 DOM 物件的機會，徹底做到「關注點分離」，強迫你必須抽象思考，用更加靈活的方式解決問題。 + 更加難能可貴的地方在於，用這種嶄新的開發架構，並不會犧牲效能與增加開發複雜度，反而得到的是高效能與徹底降低開發的複雜度等優點！ +

+
+
+ 概念: +

+ AngularJS框架核心概念是MVC(Model-View-Controller)或MVVM(Mode-View-ViewModel),主要為邏輯單元&關注點分離的一種開發方式
+ Model=資料、View=UI、Controller=工作邏輯,盡可能與其他單元分開,讓程式碼更加模組化、重用性(reusable)易維護。 + AngularJS採取宣告性質編譯方式,讓人更容易編寫與理解應用程式,可以輕易觀看HTML內容與Controller來了解程式意圖
+

+
+
+ AngularJS的六大理念 +

+

    +
  • +

    資料綁定(data-binding)

    + JQuery以直觀性方式找尋元素(DOM)並修改其內容值,屬於單向綁定(one-way-binding)
    + $('#DOM').value(text)
    + AngularJS作法以資料面(Model)來進行,每當資料發生變化時會自行處理相關UI的內容更新,屬於雙向綁定(two-way-binding)
    + < input type="text" ng-model="Demo" />{{Demo}} +
    • +
  • +

    宣告性(Declarative)

    + HTML延伸加入NG屬性,透過AngularJS裡一些Directives(指令)來達成 +
    • +
  • +

    關注點分離

    + Model:大部分為純資料,使用JSON物件呈現
    + View:使用者觀看與互動而最終描繪的HTML內容
    + Controller:不直接引用到View,固Controller與View彼此是獨立,也不須實體化 +
    • +
  • +

    相依性注入(Dependency Injection)

    + 索取Controller或Service(服務)的相依性內容之概念，而非使用new運算子做實體化或明顯呼叫函式來生成實體
    + 程式碼部分轉為負責(Injector),這樣的做法可以讓處理某段程式碼之前，馬上就可以知道我們需要什麼 +
    + angularjs.controller("ctrlA",["$scope","$log",function(scope,log){}])
    + $scope、$log都算是服務透過Injection方式,給予Controller內部使用,並變換名稱 +
    +
    • +
  • +

    擴展(Extension)

    + 可建立自訂的Directive Template(樣式範本),結合第三方套件如JQueryUI、BootStrap並給予HTML直接套用 +
    • +
  • +

    單元測試(Unit Test)

    + AngularJS每個小單元都設計成可測試,從controller、services、directives、views、routes
    + 可使用快速測試執行器(Karam)進行,http://karma-runner.github.io +
